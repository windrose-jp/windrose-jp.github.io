# Windrose スクショ採用フロー

## いつ使うか

Windrose プレイヤーが `_screenshots/` に置いた raw スクショを `images/` に採用するとき。プレイヤー個人特定の手がかりになる**右上のキャラ名表示**を消す必要がある場合のみ。

## 採用フロー

1. **画面分類**: スクショを Read して画面種別を判定
   - **メニュー系** (タブが画面上部に並んでいる) → マスク必要
     - キャラクター / インベントリ / マップ / 発見 / 冒険 / タレント / キュリオス / 進捗
   - **非メニュー系** → マスク不要 (そのまま採用 or トリミングのみ)
     - フィールドゲームプレイ、NPC 会話、ボス戦、舵輪操作、ESC ポーズ、ジャーナル単独画面など
   - **採用不可**: 他プレイヤーが映る Co-op 画面 / アルファベット文字列 (プレイヤー名候補) が映り込んでいる画面

2. **メニュー系のみマスク実行**:
   ```bash
   python D:/Code/game-wiki/windrose-jp/scripts/mask_player_name.py \
     "D:/Code/game-wiki/windrose-jp/_screenshots/Windrose ... .png" \
     "D:/Code/game-wiki/windrose-jp/images/<adopted-name>.jpg"
   ```
   - 入力: `_screenshots/` 内の raw png
   - 出力: `images/` 内の意味のある名前の jpg (例: `talent-fencer-tree.jpg`)
   - jpg にすることでファイルサイズを削減（mdbook ロード軽量化）

3. **非メニュー系**:
   - そのままコピー or 必要な範囲だけクロップ。マスクスクリプトは通さない。

4. **md に埋め込み**:
   ```markdown
   ![説明テキスト](../images/<adopted-name>.jpg)
   ```
   - パス起点は `windrose-jp/` 配下のページから見た相対パスで `../images/...`

5. **changelog.md に `[攻略]` で1行追記**:
   ```markdown
   - [攻略] スクショ追加：xxx ページに <内容> 画像
   ```

## マスク仕様（参考）

スクリプトは右上 (x: 0.742\*w 〜 w, y: 0 〜 0.066\*h) を `#1c1d21`（メニュー背景の暗灰色）で塗りつぶす。Lv ダイヤモンド枠（y=90 以降）はマスク領域外なので保持される。2560x1440 で校正済み、他解像度は比例で動く。

## やってはいけないこと

- **フィールド画面にマスクを適用しない**：HUD レイアウトが違うので不要な箇所を塗ってしまう
- **アルファベット文字列が映ったスクショを採用しない**：マスクで対応不可能、不採用にする
- **マスク済み画像を再マスクしない**：劣化と無意味
- **元の `_screenshots/` の画像を直接書き換えない**：必ず `images/` への出力で別ファイル化
